 1. WHAT THIS IS
-------------------------------------------------------------------
  Cross-platform desktop app for tracking live market data (stocks, ETFs,
  commodities, crypto, indices), aggregating financial news, monitoring FRED
  macro indicators, tracking a personal portfolio (transactions, positions,
  average-cost P&L), and running local ML (price forecasting, news sentiment,
  volatility, correlation). Single-user, local-only - all data stays on the
  machine; the only outbound calls are to free public data sources (Yahoo
  Finance / yfinance, CoinGecko, FRED). Target platforms: macOS + Windows
  (Linux deferred). Audience: the developer (Masimba) personally; also a
  portfolio/brand showcase of an AI-assisted, production-shaped desktop app.

 2. TECH STACK & ARCHITECTURE
-------------------------------------------------------------------
  TOP-LEVEL SHAPE
    Tauri v2 (Rust) shell spawns the Python sidecar as a CHILD PROCESS at
    launch, picks a random free 127.0.0.1 port, passes it via env var, waits
    for /api/health/ before showing the window, and kills the child on exit.
    React UI (in the webview) talks to the sidecar over localhost HTTP only.

  LANGUAGES / FRAMEWORKS
    - Shell: Rust (Tauri v2) - minimal, just process mgmt + plugins.
    - UI: React 19 + TypeScript + Vite + Tailwind v4 + Zustand.
    - Charts: TradingView Lightweight Charts v5.
    - Backend sidecar: FastAPI + uvicorn, Python 3.11+ (CI uses 3.13).
    - ORM/migrations: SQLAlchemy 2.x (SYNC, not async) + Alembic.
    - Scheduler: APScheduler (BackgroundScheduler, SQLAlchemyJobStore in the
      same SQLite file so jobs persist across restarts).
    - ML: statsmodels (SARIMAX, Holt-Winters/ETS), vaderSentiment, numpy/
      scipy/pandas/patsy. Lazy-imported so a sidecar without requirements-ml.txt
      still boots.
    - Packaging: PyInstaller (one-folder freeze of the sidecar) + Tauri bundler.

  WHERE FINANCIAL DATA LIVES (important)
    SQLite file, NEVER in the repo:
      - dev:      ./fintrack.db (repo root, GITIGNORED)
      - prod Mac: ~/Library/Application Support/FinTrack/fintrack.db
      - prod Win: %APPDATA%\FinTrack\fintrack.db
    WAL mode; *.db / *.db-wal / *.db-shm all gitignored. The portfolio
    transactions table (your buys/sells) lives ONLY here. There is a populated
    fintrack.db (+ -wal, -shm) in this working copy on disk - it is NOT tracked
    and must not be committed (see section 6).

  EXTERNAL SERVICES (all free, no paid tiers)
    yfinance (Yahoo) for prices + name search, CoinGecko for crypto OHLC,
    FRED for macro (needs a free API key, optional), Yahoo Finance RSS for news.
    yfinance + Yahoo RSS are UNOFFICIAL and break periodically - the ingestion
    layer is intentionally source-swappable.

  DIRECTORY LAYOUT (tracked)
    sidecar/                Python FastAPI backend
      main.py               uvicorn entrypoint; per-launch auth-token middleware
      config.py             env vars + platformdirs DB-path resolution
      api/                  routers: health, assets, prices, macro, news,
                            watchlists, alerts, config, forecast, analytics,
                            portfolio, quotes
      db/                   engine.py, models.py, base.py, seed.py,
                            migrations_runner.py, migrations/versions/0001..0014
      ingestion/            yfinance_fetcher, coingecko_fetcher, fred_fetcher,
                            rss_fetcher
      scheduler/            __init__.py (job registration), jobs.py
      services/             assets, alerts, watchlists, settings, quotes,
                            portfolio
    ml/                     forecast, persistence, jobs, accuracy, sentiment,
                            volatility, correlation
    shell/                  Tauri app
      src/                  React UI (pages/, components/, api/client.ts,
                            hooks/, layout/, stores/)
      src-tauri/            lib.rs (entry, NOT main.rs), tauri.conf.json,
                            Cargo.toml, capabilities/, icons/
      scripts/freeze-sidecar.mjs   PyInstaller wrapper fired by tauri build
    tests/                  41 pytest files
    docs/
      development/release_process.md   release runbook
      superpowers/specs/2026-06-16-forecast-uncertainty-first-design.md  (CURRENT in-flight spec)
      archive/             original pre-pivot web-app planning docs (historical)
    .claude/                project memory: PROGRESS.md (READ FIRST after this),
                            ARCHITECTURE.md, DECISIONS.md, commands/
    sidecar.spec            PyInstaller one-folder spec
    requirements*.txt       runtime / dev / ml / packaging dep ladders
    alembic.ini, pyproject.toml, CLAUDE.md, README.md, LICENSE (MIT)
    .github/workflows/      ci.yml, release.yml

  ENTRY POINTS
    - Sidecar standalone:  python -m sidecar.main  (reads FINTRACK_PORT)
    - Full app:            pnpm -C shell tauri dev  (Tauri spawns the sidecar)
    - Rust entry:          shell/src-tauri/src/lib.rs (Tauri v2 puts entry in
                           lib.rs; main.rs is a thin shim).

  CONVENTIONS / GOTCHAS (the ones that bite)
    - Sidecar binds 127.0.0.1 ONLY, random port. Never 0.0.0.0, never hardcode.
    - Per-launch auth token: shell mints a UUID, passes it to the sidecar, and
      sends it on every request via the X-FinTrack-Token header (see
      sidecar/main.py middleware + shell/src/api/client.ts). Localhost defence-
      in-depth, not a user secret.
    - SQLite: WAL + synchronous=NORMAL + foreign_keys=ON + busy_timeout=5000.
    - DB-path heuristic: cwd has pyproject.toml + sidecar/ => ./fintrack.db;
      else platformdirs app-data dir.
    - Migrations auto-run to head on sidecar startup. Current head = 0014.
    - React 19 lint rule react-hooks/set-state-in-effect: never setState
      synchronously in a useEffect body - use .then/.catch or key= remount.
    - window.confirm/alert/prompt are SILENTLY SUPPRESSED in the Tauri WKWebView
      - use the in-app ConfirmDialog component.
    - Bundle identifier is com.fintrack.desktop (was com.fintrack.app); DB is
      platformdirs-appname-keyed so it carries over.

 7. HOW TO RUN & DEVELOP
-------------------------------------------------------------------
  PREREQUISITES
    Python 3.11+ (CI: 3.13), Node 20+ with pnpm (CI: Node 22, pnpm 10),
    Rust toolchain (rustup) for Tauri, plus Tauri OS prereqs:
    https://v2.tauri.app/start/prerequisites/

  INSTALL
    python -m venv .venv
    source .venv/bin/activate                 # Windows: .venv\Scripts\activate
    pip install -r requirements.txt -r requirements-dev.txt -r requirements-ml.txt
    pnpm -C shell install

  DEV
    pnpm -C shell tauri dev                    # Tauri spawns the sidecar child
    # iterate on backend without rebuilding the shell:
    FINTRACK_PORT=8765 python -m sidecar.main
    FINTRACK_EXTERNAL_SIDECAR=1 pnpm -C shell tauri dev

  CHECKS (run all before claiming done)
    pytest
    ruff check .
    mypy --strict sidecar/ ml/
    pnpm -C shell lint
    pnpm -C shell build

  PACKAGE (unsigned, local)
    pnpm -C shell tauri:build:unsigned
    # release runbook: docs/development/release_process.md

  ENV VARS / SECRETS (names only - never commit values)
    Runtime (injected by the shell / used in dev):
      FINTRACK_PORT                 random free port the shell assigns the sidecar
      FINTRACK_DB_PATH              override DB path (tests use a tmp file)
      FINTRACK_EXTERNAL_SIDECAR=1   dev: shell won't spawn; use port 8765
      LOG_LEVEL                     INFO (DEBUG in dev)
      FINTRACK_ENABLE_SCHEDULER / FINTRACK_ENABLE_SEED   kill switches (env-only)
      FINTRACK_DISABLE_PARENT_WATCHDOG=1   disable the orphan-cleanup watchdog
      FINTRACK_SKIP_FREEZE=1        skip PyInstaller in the tauri beforeBundle step
    Data source:
      FRED_API_KEY / FINTRACK_FRED_API_KEY   free key from fred.stlouisfed.org.
        NOT required to run; without it the Macro page is empty. In the packaged
        app it is entered via Settings and stored in the local SQLite DB, not env.
    CI / release (GitHub Actions repo secrets, currently UNSET -> unsigned builds):
      APPLE_CERTIFICATE, APPLE_CERTIFICATE_PASSWORD, APPLE_SIGNING_IDENTITY,
      APPLE_ID, APPLE_PASSWORD, APPLE_TEAM_ID  (macOS signing/notarisation)
      WINDOWS_CERTIFICATE, WINDOWS_CERTIFICATE_PASSWORD  (Windows signing)
      TAURI_SIGNING_PRIVATE_KEY, TAURI_SIGNING_PRIVATE_KEY_PASSWORD  (updater)
    There is NO .env to copy; no secrets are needed for normal dev.
